1. 基本設計方針:
   - レスポンス最優先（オプティミスティックUI）を採用し、通信待ちを感じさせない爆速アプリにする。
   - Googleインフラ（認証、カレンダー、マップ、Keep/ドキュメント、スプレッドシートDB）をフル活用する設計にする。
   - 将来的なGoogle Cloud SQLへの移行を見越し、データアクセス層（DB操作ロジック）を本体ロジックから分離したアーキテクチャにすること。

2. 4本柱のモジュール構成:
   - G-LAND（個人基盤）: スコア入力、計算エンジン、個人スタッツ記録。
   - G-COMPETE（イベント）: 大会エントリー・予約管理、リアルタイムリーダーボード。
   - G-ACADEMY（教育）: 生徒カルテ管理、レッスン予約、コーチ連絡。
   - G-COURSE（運営）: ラウンド予約・受付、配車管理、決済、スタンプ・クーポン連携。

3. ログイン・認証戦略（ハイブリッド方式）:
   - Googleアカウント認証をベースにしつつ、初期利用のハードルを下げるため「ゲストログイン（ニックネームのみ）」を許可する。
   - ゲスト利用後、後からGoogleアカウント連携（データ統合）ができるシームレスな仕組みを実装すること。

4. 連携の仕組み:
   - すべてのアプリは共通のGoogleスプレッドシート（将来はSQL）をデータ源とし、Google認証でID連携させる。
   - アプリ間連携はURLパラメータ遷移で実現し、シームレスに行き来できるようにする。
   - 他の地域アプリ（スマートランドやバス停・交通情報）とも将来的に連携可能な拡張性を持たせること。

5. 作業依頼:
   - まずは「G-LAND」を単体で爆速動作するコアエンジンとして完成させ、テスト環境で動く最小構成（MVP）を提示してほしい。
   - その上で、他3つのモジュール（COMPETE, ACADEMY, COURSE）を構築するためのディレクトリ構成と連携設計図を提示すること。

【G-LAND 爆速プラットフォーム：統合指示書（要約版）】
1. 基本コンセプト
「爆速・シンプル・軽量」を最優先とし、必要時以外は通信を行わない動的UI設計とする。
画面遷移を極限まで減らし、LINEのようなオーバーレイ（重ね合わせ）表示を駆使する。
2. 主要機能構成
G-LAND（本体）: 個人のスコア管理。入力画面は常にキャッシュから即座に起動する。
G-COMPETE（コンペ管理）: ホール設定（ドラコン・ニアピン等）は絵文字アイコンで表示し、HTMLの軽さを保つ。
G-TALK（チャット）: スコア画面と連動した簡易チャット。オーバーレイで呼び出し、履歴を溜め込まない。
マルチコーチング: 複数のプロ（G-PRO）と紐付け可能。生徒側は各プロの指導カルテを統合管理できる。
3. プロ（先生）向け機能（ダッシュボード）
自動連携: 生徒のスコア入力をリアルタイムで可視化。プロは「見るだけ」で指導ポイントを把握可能。
指導カルテ: プロ専用管理画面で修正ポイントや課題を記録し、生徒と共有。
ワンタップ指導: 必要な時のみ、プロから生徒へアドバイスを送信できる機能。
4. 連携・運用仕様
LINE連携: ログインおよび通知機能として活用。LINE公式アカウントを「呼び鈴（プッシュ通知）」として使い、アプリへのジャンプ導線を整備する。API料金を考慮し、まずは無料枠内での最小構成で実装する。
QRコード連携: ユーザーおよびプロが自身のID（カルテ）を即座に共有し、紐付けを完了させる仕組み。
5. 今後の開発指針
ユーザー・プロ双方にとって「入力の手間がゼロ」になることを目指す。
データの複雑化はバックグラウンドで処理し、操作画面には「軽快さ」のみを反映させること。

# Golf DX Platform "G-LAND" 総合開発指示書

プロジェクトの目的は、単なるスコア管理アプリを超え、ゴルファーのライフサイクルを支える「7本柱のDXプラットフォーム」を構築することである。
以下の設計方針に基づき、まずはコアエンジンである「G-LAND」の再構築から着手してほしい。

## 1. 基本設計方針：爆速・軽量・拡張性
- **オプティミスティックUI**: 通信待ちを感じさせない爆速設計。入力は即座に反映し、同期はバックグラウンドで行う。
- **コンポーネント分離**: データ操作ロジックとUIを完全に分離。将来のGoogle Cloud SQL移行を前提としたDBアクセス層の設計。
- **オーバーレイUI**: 画面遷移を極限まで減らし、LINEのように「ペラっと重なる」操作感を実現する。
- **共通ナビゲーション**: 画面下部に共通フッターを配置し、常に「スコア入力・ランキング・履歴」へ瞬時にアクセスできる状態を維持する。

## 2. 7本柱のモジュール構成
- G-LAND（個人基盤）、G-COMPETE（コンペ）、G-LESSON（個人指導）、G-ACADEMY（教育）、G-COURSE（運営）、G-PRO（プロ管理）、G-TAX（経費）。
- すべてのモジュールはURLパラメータによるシームレス遷移と、Google認証によるID連携で統合する。

## 3. 今回の最優先実装（MVP: G-LAND コアエンジン）
現在のコードをベースに、以下を適用してリファクタリングせよ。
1. **共通UIパーツ化**: ナビゲーションを共通部品化し、どのページでも表示されるように修正。
2. **キャッシュ・ファースト**: スコア入力画面は、通信なしでも即座に描画される設計にすること。
3. **プロ/コーチ連携構造**: ユーザーIDに複数のプロIDを紐付け可能なデータベース構造（多対多）の実装。
4. **QRコード基盤**: ID紐付けを完了させるためのQRコード生成・読み取り導線の確保。

## 4. 連携・運用仕様
- **LINE連携**: API利用料を抑えるため、まずは「リッチメニューからの遷移」と「重要な更新通知」に絞る。
- **データ管理**: GoogleスプレッドシートをDBとして活用しつつ、将来的なSQLへのデータ移行を容易にするデータ構造を定義すること。

## 5. 作業依頼
- まずは「G-LAND」の爆速コアコード（index.html, script.js, コード.gs）を再構築して提示せよ。
- その後、他6モジュールを段階的に拡張するためのディレクトリ構造図と、モジュール間の連携データフロー図を提示すること。
